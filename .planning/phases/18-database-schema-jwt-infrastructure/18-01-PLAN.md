**Dependencies to add to package.json:**

- @types/jsonwebtoken

**JwtService Structure:**

```typescript
import { Injectable, Logger, UnauthorizedException } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import * as jwt from 'jsonwebtoken';

export interface JwtPayload {
  userId: string;
  organizationId: string;
  role: string;
  permissions: string[];
}

export interface SignOptions {
  expiresIn?: string | number;
}

export interface VerifyOptions {
  ignoreExpiration?: boolean;
}

@Injectable()
export class JwtService {
  private readonly logger = new Logger(JwtService.name);
  private readonly jwtSecret: string;

  constructor(private configService: ConfigService) {
    this.jwtSecret = this.configService.get<string>('JWT_SECRET');
    if (!this.jwtSecret) {
      throw new Error('JWT_SECRET is not defined in environment variables');
    }
  }

  sign(payload: JwtPayload, options?: SignOptions): string {
    const tokenOptions: jwt.SignOptions = {
      expiresIn: options?.expiresIn || '15m', // 15 min default for access token
    };
    return jwt.sign(payload, this.jwtSecret, tokenOptions);
  }

  verify(token: string, options?: VerifyOptions): JwtPayload {
    try {
      const decoded = jwt.verify(token, this.jwtSecret, {
        ignoreExpiration: options?.ignoreExpiration || false,
      }) as JwtPayload;
      return decoded;
    } catch (error) {
      if (error instanceof jwt.TokenExpiredError) {
        throw new UnauthorizedException('Token has expired');
      }
      if (error instanceof jwt.JsonWebTokenError) {
        throw new UnauthorizedException(`Invalid token: ${error.message}`);
      }
      throw new UnauthorizedException('Token verification failed');
    }
  }

  signRefreshToken(payload: JwtPayload): string {
    return this.sign(payload, { expiresIn: '7d' });
  }

  signAccessToken(payload: JwtPayload): string {
    return this.sign(payload, { expiresIn: '15m' });
  }
}
```

**Implementation Notes:**
- Load JWT_SECRET in constructor; throw error if missing (fail-fast)
- Use jsonwebtoken library (jwt.sign, jwt.verify) directly
- Return typed JwtPayload from verify()
- Handle three error cases: TokenExpiredError, JsonWebTokenError, generic error
- Provide convenience methods signAccessToken() and signRefreshToken() for Phase 19/22
- No database interaction in JwtService (pure crypto)
- Access token expiry: 15 minutes (short-lived)
- Refresh token expiry: 7 days (long-lived, will store in DB in Phase 22)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Unauthenticated POST /auth/signup receives email + password (requires HTTPS in production) |
| API → Database | SQL injection via Prisma (ORM prevents raw SQL injection; parameterized queries) |
| Environment → App | JWT_SECRET injected via environment variables (trusted if env is secure) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Spoofing | JwtService | Mitigate | Use strong JWT_SECRET (min 32 chars), validate JWT_SECRET at app startup (fail-fast) |
| T-18-02 | Tampering | Database schema | Accept | PostgreSQL enforces constraints; org isolation via FK/unique constraints; no app-level bypass |
| T-18-03 | Information Disclosure | JWT token | Accept | Phase 19 will enforce HTTPS; JWT tokens sent in response (acceptable for now, no sensitive data in v1 tokens beyond userId/orgId) |
| T-18-04 | Denial of Service | JwtService | Accept | No rate limiting in Phase 18 (JWT verify is fast); Phase 21 will add guard rate limits |
| T-18-05 | Elevation of Privilege | Role field | Mitigate | Role stored as text with CHECK constraint; Prisma validates on write; Phase 21 RoleGuard enforces permission checks |
| T-18-06 | Repudiation | JWT signing | Accept | JwtService signs all tokens with shared secret; audit trail via created_at timestamps in users/organizations tables (Phase 22 logging) |

**Summary:** Phase 18 is infrastructure-only (no API endpoints). Biggest risk is JWT_SECRET exposure (mitigated by secure env storage and fail-fast validation). Database constraints prevent schema-level violations. Phase 19+ will add rate limiting, HTTPS enforcement, and permission guards.
